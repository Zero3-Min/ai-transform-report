# 案例：陕西广电网络"秦岭云"——本报告推荐技术栈的省级生产环境实践（厂商披露案例）

> **利益相关提示（引用前必读）**：本案例被 MaxKB 厂商（飞致云/FIT2CLOUD）列入其官方应用案例，主要传播渠道为厂商与机构自有账号及行业媒体转发，**属厂商披露案例**——效果表述可能偏正面、未经独立第三方核验。本案例库收录它的目的，是佐证本报告第 6 章所推荐开源平台的**生产环境成熟度**，而非佐证任何效果数字。

## 公司背景

陕西广电网络传媒（集团）股份有限公司为深交所上市公司（600831，"广电网络"），"秦岭云"是其融媒体/智慧广电技术平台。**注意规模差异**：这是一家省级上市公司，人员与预算规模远超本报告 10–200 人的目标读者，本案例的参照价值在技术路线而非投入规模。
[来源: 广电独家（tvoao.com）, 《重磅！陕西广电网络"秦岭云"平台实现DeepSeek本地化部署》, 2025年2月, https://www.tvoao.com/a/220463.aspx]

## 做了什么

2025 年 2 月，"秦岭云"平台完成 DeepSeek 大模型的本地化部署，将其应用于新闻生产与服务场景，在数据安全与功能适配上跑通了"开源大模型 + 广电业务平台"的组合；MaxKB 官方将"陕西广电网络'秦岭云'平台实现 DeepSeek 本地化部署"列为其应用案例，即该组合中知识库/智能体层采用 MaxKB 开源平台。
[来源: 广电独家（tvoao.com）, 同上, 2025年2月]
[来源: MaxKB 官方网站应用案例（厂商披露）, 检索日期2026年7月, https://maxkb.cn/]

## 用了什么技术

DeepSeek 大模型本地化部署 + MaxKB 开源知识库/智能体平台（飞致云出品）。这一组合与本报告第 6 章为预算敏感型客户推荐的技术栈（MaxKB + 已备案国产大模型）完全同构，区别仅在部署路径：省级机构因数据安全要求选择本地化部署（对应第 6.3 节"私有化路径"），而多数 10–200 人公司选纯 API 模式即可。

## 量化效果

**未见任何量化运营数据披露**（截至本次调研检索）。厂商与行业媒体报道均为部署完成与应用方向的描述，无产量、效率、成本类指标。

## 对 10–200 人小公司的适配启示

- **平台成熟度佐证，而非规模参照**：省级上市广电机构在生产环境中采用 MaxKB + DeepSeek 组合，说明本报告第 6 章推荐的开源技术栈不是"玩具方案"——连数据安全要求严苛的国有媒体平台也在用同一开源底座。小公司借鉴的是"这套开源组合可堪生产之用"这一事实，而非其部署规模。
- **部署路径的分野印证第 6.3 节的判断**：省级机构选本地化部署，是因为其数据安全与监管要求必须"数据不出内网"；10–200 人公司多数场景用纯 API 模式（无 GPU 投入）即可满足合规与机密隔离需求，不必照搬本地化路径的硬件成本。
- **厂商案例要打折听**：本案例经厂商渠道披露且无量化数据，与案例 04（天下秀灵感岛）的启示相同——采购决策应以贵司真实场景实测为准，而非凭"某某大机构也在用"的品牌背书。本报告第 6 章的选型对比（含各平台短板与权限颗粒度差异）正是为替代品牌背书的实测框架而写。
