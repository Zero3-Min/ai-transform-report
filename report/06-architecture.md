# 技术架构与选型

> **本章决策要点**：预算有限、优先低成本起步的企业，建议选用 MaxKB，云端月成本约 100–200 元；预算相对充足、需完整部署创作流水线的企业，建议选用 Dify，云端月成本数百元。以下为技术选型的具体依据，供技术负责人评估参考；管理层可直接参考本结论，并查阅 6.2“部署”条目的分档成本或第 11 章的整体服务报价。

### 6.1 分层架构

```
应用层   问答界面 / 写作助手 / 预审插件（飞书、企微、浏览器内嵌）
编排层   Dify / FastGPT（工作流、Agent、提示词版本管理）
知识层   向量检索 + 关键词混合检索；文档解析（RAGFlow 可选，见6.2）
模型层   模型路由中间层：国产合规大模型（对外内容）+ 可替换设计
数据层   私有化存储 / 国内云；权限、日志、脱敏
```

### 6.2 选型建议

- **平台（分预算敏感型/标准型两档）**：四个主流开源平台并非单纯的“功能多少”之分，而是各有侧重——Dify（应用开发全能，工作流与 Agent 能力最强）、MaxKB（开箱即用最简单，部署门槛最低）、FastGPT（知识库优先，社区版即具备细粒度权限）、RAGFlow（文档理解最强，但硬件门槛也最高）。**预算敏感型**（团队规模较小、优先低成本起步）推荐 MaxKB——部署门槛四者最低，且提供“问答→工作流→Agent”的渐进式升级路径，与本报告第 4 章三层递进蓝图吻合，不需要中途换平台。**标准型**（预算相对充足、目标是完整跑通阶段 1–2）推荐 Dify 为主平台，社区活跃度断层领先，生态插件丰富，出问题更容易找到解决方案；若客户档案库权限隔离要求严格，可用 FastGPT 替代 Dify 作为主平台（FastGPT 社区版即内置精细的角色权限设置，可按“读取、编辑、管理”分级控制，即 RBAC 权限体系，权限粒度优于 Dify、MaxKB 的社区版）[来源: FastGPT 官方文档, 《团队&成员组&权限》, 官方文档持续更新无固定发布日期/检索日期2026年7月, https://doc.fastgpt.cn/zh-CN/guide/workspace/team/team_roles_permissions]。
- **开源协议提示**（要点：四平台均可免费商用，但对二次包装对外销售有不同限制）：Dify、FastGPT 的使用条款规定，不得将其改造后作为自有付费产品向多个外部客户提供服务（即“多租户 SaaS”），但企业内部单租户使用不受此限；MaxKB 采用 GPL v3 协议，其额外义务仅在“对外分发修改后的代码”时触发，内部部署使用不受影响；RAGFlow 采用的协议限制最少。**唯一例外**：若企业未来拟开展第 4 章所述“将系统产品化对外服务”业务，需提前购买 Dify 商业授权，或改用限制最少的 RAGFlow。
- **知识层文档解析（RAGFlow 何时引入）**：RAGFlow 在复杂文档解析上具备优势，可识别扫描件文字、复杂表格与版面结构（即 DeepDoc 深度文档理解），能力明显强于 Dify、MaxKB 的基础文本抽取，但对服务器配置要求更高（官方建议至少 32GB 内存），部署成本亦为四者中最高。**仅当客户档案库或爆款拆解库中存在较多复杂 PDF、扫描件或含复杂表格的文档时，才建议引入 RAGFlow 做知识层增强；如客户资料以规范化文本为主，Dify/MaxKB 自带的基础解析已经够用**，避免不必要的硬件投入。
- **模型策略**：对外发布内容优先选用已备案的国产大模型（合规稳妥）；内部提效场景可混用。中间层使模型可替换，避免锁定。浙江瑞安市融媒体中心 2025 年 2 月直接接入 DeepSeek-R1 API（非自建模型）上线“瑞融AI”，验证了本报告“接入已备案国产大模型、不自研底层模型”这一路线在小规模机构中同样可行。详见 [cases/case-03-瑞安市融媒体中心.md](../cases/case-03-瑞安市融媒体中心.md)。
- **部署（分平台成本估算）**：≤50 人团队可云端托管起步，但月成本因平台而异——MaxKB/Dify 最低配置（2核4GB）云主机月成本约 100–200 元级；FastGPT 推荐配置（4核16GB）约 400–600 元级；RAGFlow 因内存需求高（32GB起）月成本可能超过 800 元，是 MaxKB 方案的 4–6 倍（**注**：以上为基于通用云主机定价惯例的数量级估算，非平台官方发布数据，正式报价前建议按当前云厂商挂牌价复核）；涉及客户机密档案时上私有化（单台 GPU 服务器或纯 API 模式 + 私有数据库均可）。
- **权限**：客户档案库按商务线隔离；合规红线库全员只读；写权限仅责任人。若选用 Dify/MaxKB 作为主平台，社区版权限颗粒度较粗，细粒度角色拆分（如创建与复制权限分离）需付费版本；若权限隔离是硬性要求，优先考虑 FastGPT 或在预算中预留平台付费升级空间。

> **案例参考**：并非所有预算敏感型客户都需要自建知识库平台。上市公司天下秀推出的 AIGC 营销工具“灵感岛”，面向中小 MCN 与门店提供开箱即用的内容生成、数字人克隆等能力，已为超 2 万家门店完成数字人克隆。对完全不具备自建能力的公司，采购成熟第三方工具可作为比自建更快见效的过渡方案；但需注意灵感岛类创新业务收入占比目前不足 5%，采购前应实测效果而非仅凭平台品牌决策。详见 [cases/case-04-天下秀.md](../cases/case-04-天下秀.md)。

### 6.3 可行性关键问题说明（选型时效、私有化成本、迁移成本、数据安全）

- **选型时效性**：本报告选型依据截至 2026 年 7 月的各平台版本（详见 `research/03-技术选型.md`）。开源平台迭代快，建议**每半年复评一次**主流平台能力与版本。本架构以"编排层/模型层可替换"为设计前提——即便某平台被更优者取代，替换的是编排层，知识库数据与业务流程可迁移保留，不必推倒重来。
- **私有化的真实硬件成本**：私有化分两种路径，成本差异大：（1）**纯 API 模式**——本地只部署知识库与编排层、大模型仍调用已备案的国产大模型 API，无需 GPU，普通服务器或云主机（约 4 核 16GB）即可，一次性硬件成本或年租数千元至万元级；（2）**本地部署开源模型**——需 GPU 服务器（如单张 A10/4090 级显卡），一次性硬件成本约 2–5 万元（按配置浮动），另需运维投入。**多数 10–200 人传媒公司选路径（1）即可满足合规与机密隔离需求**，无需承担 GPU 成本（此为方向性估算，正式采购前按当期硬件报价核算）。
- **模型替换的实际迁移成本**：中间层使模型调用可切换，但迁移**并非零成本**——更换模型后需重新调优提示词、并对照固定评测集重跑效果回归测试（约数人日工作量），因为不同模型对同一提示词的响应存在差异。故本方案建议以"双模型热备 + 灰度对比"方式迁移，先小流量验证新模型效果不低于旧模型，再全量切换，避免"换模型即掉质量"。
- **数据安全承诺的技术兑现方式**：数据安全不是口头承诺，对应具体技术措施——传输层 TLS 加密；存储层对客户档案库等敏感数据加密；权限层客户档案库按商务线做 RBAC 隔离、合规红线库全员只读、写权限仅责任人；审计层保留完整访问与操作日志（建议留存不少于 6 个月，兼顾第 7 章合规举证需要）；入库前对客户机密信息脱敏；选择私有化路径时数据不出客户内网。上述措施在阶段 1 部署时逐项配置并向客户验收。
