# 技术架构与选型

### 6.1 分层架构

```
应用层   问答界面 / 写作助手 / 预审插件（飞书、企微、浏览器内嵌）
编排层   Dify / FastGPT（工作流、Agent、提示词版本管理）
知识层   向量检索 + 关键词混合检索；文档解析（RAGFlow 可选）
模型层   模型路由中间层：国产合规大模型（对外内容）+ 可替换设计
数据层   私有化存储 / 国内云；权限、日志、脱敏
```

### 6.2 选型建议

- **平台**：Dify（功能全、工作流强、生态好，适合从知识库长到创作流水线）；预算极敏感时 MaxKB 起步。
- **模型策略**：对外发布内容优先选用已备案的国产大模型（合规稳妥）；内部提效场景可混用。中间层保证模型可替换，避免锁定。
- **部署**：≤50 人团队可云端托管起步（月成本数百元级）；涉及客户机密档案时上私有化（单台 GPU 服务器或纯 API 模式 + 私有数据库均可）。
- **权限**：客户档案库按商务线隔离；合规红线库全员只读；写权限仅责任人。
