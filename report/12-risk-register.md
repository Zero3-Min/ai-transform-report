# 风险登记册

| 风险 | 概率 | 影响 | 应对 |
|---|---|---|---|
| AI 生成内容引发合规事故 | 中 | 高 | 人工终审强制化 + 预审拦截 + 全程留痕，写入合同双方责任边界 |
| 知识更新停滞导致系统荒废 | 高 | 高 | 周更责任制 + 年度运营服务 + 月复盘通报更新量 |
| 员工抵触或私下绕开系统 | 中 | 中 | 种子用户共建、KPI 重定义、"省时间"正反馈通报 |
| 客户期望"全自动"落空 | 中 | 中 | 阶段 0 签署量化验收确认书，宣传口径统一为"人机协同" |
| 大模型 API 涨价或停服 | 低 | 中 | 模型路由中间层，双模型热备 |
| 客户机密档案泄露 | 低 | 高 | 私有化选项、权限隔离、脱敏入库、访问日志 |
| 技术投入脱离核心业务健康度 | 中 | 中 | 阶段 0 诊断聚焦客户核心业务健康度，P0/P1 场景优先于追逐数字人直播等热点技术，避免"寄望 AI 挽救结构性问题业务"（详见第4章遥望科技案例：头部 MCN 布局 AI 数字人，但净利润口径四年亏损超 30 亿元，技术投入未能对冲核心业务问题） |
