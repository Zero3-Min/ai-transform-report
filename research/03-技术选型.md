# 开源 AI 知识库/应用平台最新能力对比（Dify / MaxKB / FastGPT / RAGFlow）

## 调研结论摘要

1. **四个平台定位并非完全重叠，而是各有侧重**：Dify 是最全面的"应用开发平台"（工作流+Agent+LLMOps一体）；FastGPT 是"知识库优先"的轻量问答系统；RAGFlow 是"文档理解优先"的检索引擎，文档解析质量最强但硬件门槛也最高；MaxKB 是部署最简单、中文生态最友好的"开箱即用"平台。四者可以组合使用，而非互斥选择。
2. **"多租户 SaaS 限制"对本报告目标客户基本不构成实际障碍**：Dify、FastGPT 的开源协议均禁止"未经书面授权运营类似的多租户 SaaS 服务"，但这一限制针对的是"把开源版本包装成对外销售的 SaaS 产品"，10–200 人传媒公司作为内部工具使用（单租户），不受此限制影响；只有当公司自身要做第 4 章场景组合中 P3 级"AI 内容中台对外服务"时，才需要重新评估是否要购买企业版授权。
3. **MaxKB 的 GPL v3 协议"传染性"对内部使用同样不构成障碍**：GPL 的开源义务在"对外分发/销售修改后的软件"时触发，公司仅内部部署使用、不将修改后的 MaxKB 代码对外分发，无需公开自己的定制代码，无需担心"传染"到公司其他系统。
4. **硬件门槛差异显著，直接影响部署成本**：MaxKB 最低配置 2 核 4GB 即可在线部署，是四者中门槛最低的；RAGFlow 官方建议至少 32GB 内存（因需同时运行 Elasticsearch + InfinityDB），是四者中门槛最高的，云端月成本可能是 MaxKB 方案的数倍。
5. **社区活跃度上 Dify 断层领先，RAGFlow 次之**：GitHub Star 数 Dify（14.6万）>> RAGFlow（8.4万）> FastGPT（2.9万）> MaxKB（2万），四者近半年均有持续版本迭代，非"停更"项目，均可作为生产环境的候选。

### 二、文档解析质量

- **RAGFlow（最强）**：采用自研的 DeepDoc 视觉文档理解系统，集成 OCR、表格结构识别、版面分析，能在内容进入向量库前保留原始语义结构；支持可视化分块与人工干预、关键引用追溯；2026 年新增 SoMark OCR 解析器，可提取复杂表格与图表元素。支持格式最全：PDF/DOC/DOCX/TXT/MD/MDX、CSV/XLSX/XLS、JPEG/PNG/TIF/GIF、PPT/PPTX。
  [来源: infiniflow/ragflow GitHub README 及 Release Notes, 检索日期2026年7月, https://github.com/infiniflow/ragflow]

- **FastGPT（较强，需额外配置）**：社区版自 v4.9.0 起支持接入 Marker、MinerU 做增强 PDF 解析，MinerU 基于 YOLO、PaddleOCR 与表格识别模型，能提取图片、表格、公式等复杂内容；基础版本可处理 txt/md/html/pdf/docx/pptx/csv/xlsx 常见格式。
  [来源: FastGPT 官方文档, 《Integrating MinerU PDF Parsing》, 检索日期2026年7月, https://doc.fastgpt.io/en/self-host/custom-models/mineru]

- **MaxKB（基础可用）**：支持常见文档格式解析，分段预览支持 Markdown 渲染；近期版本修复了含特殊字符 PDF 的解析报错问题，但未见公开的深度版面识别或复杂表格提取能力说明，解析深度弱于 RAGFlow 与增强配置后的 FastGPT。
  [来源: 1Panel-dev/MaxKB GitHub Releases 页面, 检索日期2026年7月, https://github.com/1Panel-dev/MaxKB/releases]

- **Dify（基础，依赖生态插件）**：内置文本抽取覆盖 PDF、PPT 等常见格式，官方定位为"开箱即用"的基础能力；复杂文档解析更依赖其插件生态或与外部解析工具（如 RAGFlow）组合使用，这也是本报告现有第 6 章"文档解析（RAGFlow 可选）"这一设计的技术依据。
  [来源: langgenius/dify GitHub README, 检索日期2026年7月, https://github.com/langgenius/dify]

## 针对"10–200 人传媒公司"的选型建议

### 预算敏感型（团队规模较小、优先低成本起步、以知识库/问答场景为主）

**推荐：MaxKB**

理由：
- 硬件门槛四者最低（2核4GB即可在线部署），云主机月成本约为 RAGFlow 方案的 1/4–1/6，最贴合"数万元级投入"的预算约束（呼应本报告第 2 章的判断）。
- 提供"问答（RAG）→ Workflow → Agent"的渐进式升级路径，与本报告第 4 章"知识底座 → 流程再造 → 业务重塑"三层递进蓝图高度吻合，不需要中途更换平台。
- 中文文档与技术社区支持充分，对不具备资深技术团队的中小传媒公司更友好。
- GPL v3 协议的"传染性"义务仅在对外分发修改代码时触发，公司内部部署使用不受影响，可放心采用。
- **局限**：细粒度权限管理（如按角色拆分创建/复制权限）需付费 X-Pack，社区版权限颗粒度较粗，若客户档案隔离要求非常严格，需在阶段 1 评估是否需要升级付费版本或改用 FastGPT。

### 标准型（预算相对充足、目标是完整跑通阶段 1–2、需要严格权限隔离或计划做流水线深度定制）

**推荐：Dify 为主平台，复杂文档解析场景搭配 RAGFlow**

理由：
- Dify 的 Agent 能力（Function Calling/ReAct + 50+ 内置工具）与工作流可视化编排能力四者最强，最适合支撑本报告第 5 章"选题助手→初稿Agent→预审Agent→人工终审→发布检查"的完整创作流水线。
- 社区活跃度断层领先（14.6万 Star），生态插件丰富，出问题时更容易找到解决方案或现成组件，降低实施风险。
- 若客户档案、爆款拆解等资料以复杂 PDF/扫描件/含表格文档为主，可在知识层引入 RAGFlow 做深度文档解析（DeepDoc），弥补 Dify 基础文档解析能力的不足——这正是本报告现有第 6 章"文档解析（RAGFlow 可选）"设计的依据，本次调研进一步验证了这一组合的合理性。
- 权限方面，若需要严格的多角色权限隔离，FastGPT 的社区版权限体系比 Dify 社区版更细（位运算级 RBAC，读/写/管理三级），可作为 Dify 的替代主平台，尤其适合对客户数据隔离要求高的代运营类公司。
- **注意**：若客户未来计划将 AI 能力对外产品化（第 4 章 P3 级场景"AI 内容中台对外服务"），Dify/FastGPT 社区版的"禁止多租户 SaaS"条款将成为实际限制，需要提前预算 Dify 企业版授权费用或评估 RAGFlow（无此限制）作为替代方案。

## 对报告第 6 章（技术架构与选型）的修改建议清单

当前 `report/06-architecture.md` 的选型建议较为简略（"Dify 为主、预算敏感时 MaxKB 起步"），本次调研验证了这一方向基本正确，但可以更精确、更有数据支撑：

1. **6.2 "平台"条目**：建议补充"预算敏感型/标准型"两档选型逻辑（详见上文），并注明四平台的核心差异——不是单纯的"功能多少"，而是"Dify（应用开发全能）vs MaxKB（开箱即用最简单）vs FastGPT（知识库+细粒度权限）vs RAGFlow（文档理解最强但硬件重）"，帮助读者按自身场景对号入座，而非仅凭"预算是否敏感"单一维度决策。

2. **6.2 "平台"条目补充协议提示**：建议明确注明——MaxKB 为 GPL v3（内部使用不受"传染性"影响）、Dify/FastGPT 均限制"未经授权的多租户 SaaS 运营"（内部单租户使用不受影响，但若后续做第 4 章 P3 级"AI内容中台对外服务"需重新评估）、RAGFlow 为标准 Apache 2.0（限制最少）。避免客户在后期因协议问题产生合规疑虑。

3. **6.1 "知识层"条目**：现有表述"文档解析（RAGFlow 可选）"建议补充依据——RAGFlow 的 DeepDoc 深度文档理解能力（OCR+表格识别+版面分析）显著强于 Dify/MaxKB 的基础文本抽取，但其部署门槛也四者最高（官方建议 32GB 内存），建议明确"仅当客户档案/爆款库中存在大量复杂 PDF、扫描件或含复杂表格的文档时，才建议引入 RAGFlow 做知识层增强，否则 Dify/MaxKB 自带的基础解析已经够用"，避免不必要的硬件投入。

4. **6.2 "部署"条目**：现有表述"≤50人团队可云端托管起步（月成本数百元级）"建议按平台细化——MaxKB/Dify 最低配置云主机成本约 100–200 元/月量级，FastGPT 推荐配置约 400–600 元/月量级，RAGFlow 因内存需求高（32GB起）成本可能超过 800 元/月，四者成本差异达 4–6 倍，直接影响轻量版/标准版报价的技术可行性边界，建议第 11 章报价与此处保持口径一致（**注**：此为基于通用云主机定价惯例的数量级估算，非平台官方数据，写入正文前建议按当前云厂商挂牌价复核）。

5. **6.2 "权限"条目**：建议补充"如需社区版即具备细粒度角色权限（读/写/管理三级、按部门/群组隔离），FastGPT 优于 Dify/MaxKB 社区版"这一具体结论，为"客户档案库按商务线隔离"的设计提供更明确的平台落地路径。
